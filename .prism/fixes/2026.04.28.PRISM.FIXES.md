## Assistant

Yes — in principle, you’re right.

If `/audit`, `/fix`, etc. are “just commands,” then an LLM agent should be able to invoke the same underlying behavior without you manually typing the slash command in the terminal UI.

What’s probably happening is:

1. **Those slash commands are UI-level shortcuts**
   - e.g. your terminal/chat app intercepts `/audit`
   - and routes it to a special built-in workflow
   - instead of sending it as normal shell text

2. **The agent likely does not automatically get that same slash-command router**
   - so when *you* type `/audit`, the app knows what to do
   - but when the model acts, it may only have:
     - file access
     - shell/tool access
     - normal command execution
   - not access to the app’s proprietary “slash command dispatcher”

3. **So the issue is not “typing”**
   - it’s **capability binding**
   - `/audit` may *look* like a command, but it may actually be a **frontend command**, not a real executable on `$PATH`

### Quick test

In your terminal, run:

```bash
which audit
which /audit
type audit
type /audit
```

If those fail, then `/audit` is almost certainly **not a real shell
