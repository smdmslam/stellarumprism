# VC Pitch Deck Skills Index

This index groups the `vc-pitchdeck-` skill family so it is easy to browse, load, and use as a coherent system inside Prism.

All related files intentionally share the same filename prefix so they cluster naturally in `.prism/skills/` listings.

---

## Purpose

These skills were created to preserve a successful process for turning technical product materials into strong investor-facing pitch decks.

The goal is not only to preserve one finished deck.
The goal is to preserve the reusable method:
- how to think about the deck
- how to structure the work
- how to sharpen the language
- how to avoid weak patterns
- how to present the finished slides aloud

This skill family should be used whenever creating, revising, or pressure-testing a VC pitch deck from source materials.

---

## Skill family

### 1. `vc-pitchdeck-framing.md`
**Use for:** the core strategic reframing.

**What it contains:**
- the central principle of moving from product description to company value
- wedge → platform → moat logic
- investor framing rules
- prompting patterns that improve deck quality

**Use this when:**
- the current deck is too descriptive
- the company sounds smaller than it is
- the founder wants a more investor-oriented narrative

---

### 2. `vc-pitchdeck-framing-example-prism.md`
**Use for:** the concrete case study.

**What it contains:**
- how the Prism deck improved in practice
- what founder prompts mattered most
- approximate attribution between prompting, model capability, and Prism's harness
- the exact narrative shift that made the deck stronger

**Use this when:**
- you want to remember what worked in the Prism case
- you want an example of the general framing rules in action
- you want to preserve process lessons, not just abstract advice

---

### 3. `vc-pitchdeck-workflow-checklist.md`
**Use for:** process discipline.

**What it contains:**
- step-by-step workflow from source gathering to final readiness check
- decision points before drafting
- practical checklists for each phase

**Use this when:**
- starting a new deck from scratch
- rewriting a weak existing deck
- ensuring the work is done in the right order

---

### 4. `vc-pitchdeck-investor-language.md`
**Use for:** improving phrasing and diction.

**What it contains:**
- preferred investor concepts and vocabulary
- stronger phrasing patterns
- contrast patterns
- weak vs strong wording upgrades
- anti-pattern language to delete

**Use this when:**
- the draft sounds generic
- the language is accurate but not memorable
- the founder wants sharper investor-facing wording

---

### 5. `vc-pitchdeck-slide-patterns.md`
**Use for:** slide-by-slide structure.

**What it contains:**
- reusable slide formulas
- the persuasive job of each slide type
- what to include and what to avoid on each common deck slide

**Use this when:**
- building slide order
- deciding what each slide should do
- turning a narrative spine into an actual deck

---

### 6. `vc-pitchdeck-redflags.md`
**Use for:** negative filtering and review.

**What it contains:**
- common signs that a deck feels weak, generic, too technical, unbelievable, or forgettable
- why those failures hurt
- how to fix them

**Use this when:**
- reviewing a draft
- editing a deck that feels “off” without knowing why
- doing a final quality pass before calling the deck done

---

### 7. `vc-pitchdeck-speaker-notes.md`
**Use for:** spoken presentation quality.

**What it contains:**
- how to turn sparse slides into a compelling spoken narrative
- slide-by-slide note guidance
- what belongs in notes vs on slides
- delivery discipline and transition guidance

**Use this when:**
- preparing to present live
- creating speaker notes for the founder
- translating a good deck into a strong verbal pitch

---

## Recommended order of use

A strong default sequence is:

1. `vc-pitchdeck-workflow-checklist.md`
   - gather source material and diagnose the current deck
2. `vc-pitchdeck-framing.md`
   - define the strategic narrative spine
3. `vc-pitchdeck-framing-example-prism.md`
   - sanity-check the method against the concrete Prism case
4. `vc-pitchdeck-investor-language.md`
   - sharpen wording and investor diction
5. `vc-pitchdeck-slide-patterns.md`
   - map the narrative spine into actual slides
6. `vc-pitchdeck-redflags.md`
   - review for weakness, vagueness, or mis-sequencing
7. `vc-pitchdeck-speaker-notes.md`
   - prepare spoken delivery and presentation flow

This sequence can be compressed, but the order is intentionally logical.

## Suggested invocation patterns

These skills can be used explicitly or implicitly.

### Explicit use
Example:
> Use `vc-pitchdeck-framing.md`, `vc-pitchdeck-slide-patterns.md`, and `vc-pitchdeck-redflags.md` to rewrite this deck from the docs folder.

### Example-led use
Example:
> Read `vc-pitchdeck-framing-example-prism.md` and use that process to improve this new deck.

### Review-first use
Example:
> Audit this deck using `vc-pitchdeck-redflags.md` and then rewrite it using `vc-pitchdeck-investor-language.md`.

### End-to-end use
Example:
> Use the `vc-pitchdeck-` skill family to create a new investor deck from the current product docs and produce both slides and speaker notes.
