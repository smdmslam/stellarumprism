# Recent commits — loose ends to clean up
Date: 2026-04-28
Scope: commits `db5c9d8` (panel-split rebuild) through `25364e5` (HEAD), about 18 commits.
Combined catches from two independent reviews (Oz + Prism). Ordered by impact.

## Background
The panel-split rebuild moved agent dialogue out of xterm into the new HTML
`AgentView`. xterm is now reserved for shell I/O. Most of the commits since
then were UI polish (toolbar toggles, font-size controls, tab context menu,
etc.). Both reviews found that the architectural rule "xterm = shell only"
is broken in several places, and that one of the new menu items
(Duplicate Full Session) is half-built.

## High impact

### 1. Slash-command output leaks into the xterm shell pane
`workspace.ts` still has ~76 `this.term.write(...)` / `this.term.writeln(...)`
calls for agent-flavored messages. After the panel split these now render
in the bottom 200px shell strip instead of the agent panel.

Affected commands and lifecycle messages:
* `/help`, `/load`, `/save`, `/last`, `/model`, `/verify`
* `/audit` and `/build` report renders (`renderAnsiFindings`, `renderAnsiBuildReport`)
* file-save echo (`[edit] saved <path>`)
* `[router]`, `[agent]` notices that workspace.ts emits directly
* `[duplicate]` failure path from the new `duplicateSession()`
* error fallbacks (e.g. `[load] dialog failed`, `[save] failed`)

Fix:
* Route every agent-flavored line through `this.agentView.appendNotice(kind, body)`
  or `appendError(message)` rather than `this.term.write`.
* Keep only true shell-lifecycle writes in xterm:
  * `Failed to start shell: ...` startup error
  * `[shell exited]` on PTY exit
* For ANSI-rich rendered output (`renderAnsiFindings`, `renderAnsiBuildReport`,
  `renderHelpAnsi`, `renderModelListAnsi`, `renderHistoryAnsi`,
  `this.renderLastAnsi()`): either
  * (a) strip ANSI inside `appendNotice` (already does this) and let the
    panel render plain text, or
  * (b) convert these renderers to emit Markdown / DOM and render via
    `appendProse` so headings, code, and color land properly.
  Recommend (b) for `/help`, `/last`, `/model`, and the audit/build
  reports because their value is structural; (a) is fine for short
  notices like `[verify] off`.

Acceptance check:
* `grep -n "this\\.term\\.write\\|this\\.term\\.writeln" src/workspace.ts`
  returns ONLY the two shell-lifecycle lines listed above.

### 2. "Duplicate Full Session" is half-built
Commit `25364e5` introduced a tab context menu with a "Duplicate Full
Session" entry. Implementation:
* `Workspace.duplicateSession()` writes a temp markdown file via
  `save_chat_markdown` and returns
  `{ cwd, title, loadChatPath } as WorkspaceRestoreOptions`.
* `TabManager.newTab(restore)` instantiates a new `Workspace` with that
  restore object.
* `Workspace` constructor reads `restore.id`, `restore.cwd`, `restore.title`
  but NEVER reads `restore.loadChatPath`.

Result: the menu item silently does the same thing as "Duplicate Path /
New Chat" except it leaves an orphan temp file behind.

Fix:
* In `Workspace.init()` (or right after `agent` is constructed), if
  `this.restore.loadChatPath` is set:
  * `await this.loadSavedChat(this.restore.loadChatPath)`
  * Then delete the temp file so it doesn't accumulate (see #4).
* Surface a one-line notice in the agent panel: "Duplicated session from
  prior tab".

Acceptance check:
* Right-click a tab with chat history, choose "Duplicate Full Session".
* New tab opens with the same prompt + reply history visible in the
  agent panel.
* The `temp-dup-XXXX.full.md` file is gone after the load.

### 3. `expandTilde` is a stub
`workspace.ts:3576` (approx). Function signature claims to expand `~/`
but the body returns the input unchanged with a comment "best we can do
without a dedicated command".

Callers that actually depend on expansion:
* `loadChat()` default-path fallback when there is no cwd.
* `duplicateSession()` temp file path
  (`~/Documents/Prism/Chats/temp-dup-XXXX.full.md`).

The first call goes through Tauri's open dialog which expands `~` for us,
so it works by luck. The second writes the path through `save_chat_markdown`
without going through a dialog — depending on what the Rust handler does
with the literal `~`, either:
* Tauri / dialog APIs expand it (works), or
* a literal `~/` directory gets created under cwd (silent bug).

Fix:
* Add a Tauri command on the Rust side: `expand_tilde(path: String) -> String`
  using `dirs::home_dir()` or equivalent.
* Make the frontend `expandTilde` call that command (async).
  * Or: expose `homeDir()` once at workspace init, cache it, do string
    replacement on the JS side.
* Replace the existing stub with the working implementation.

Acceptance check:
* `await expandTilde("~/foo")` returns `/Users/<me>/foo`.
* "Duplicate Full Session" temp file lands under
  `/Users/<me>/Documents/Prism/Chats/`, not under cwd.

## Medium impact

### 4. Temp dup files are never cleaned up
Each "Duplicate Full Session" leaves
`~/Documents/Prism/Chats/temp-dup-XXXX.full.md` on disk.

Fix:
* After the new tab successfully loads the chat (see #2), delete the
  temp file via a Tauri `remove_file` command.
* Belt-and-braces: on app startup, sweep
  `~/Documents/Prism/Chats/` for `temp-dup-*.full.md` older than 24h
  and delete them. Catches the case where a duplicate failed mid-load.

Acceptance check:
* Run "Duplicate Full Session" 5 times. After all five, the chats dir
  contains zero `temp-dup-*` files.

### 5. Missing `typecheck` script in `package.json`
Prism's review found `pnpm typecheck` returns
`Command "typecheck" not found`. The build script (`tsc && vite build`)
covers it indirectly, but the explicit script is missing.

Fix:
* Add to `package.json` scripts:
  ```json
  "typecheck": "tsc --noEmit"
  ```
* Document in README / contributor docs.

Acceptance check:
* `pnpm typecheck` exits 0 on a clean tree.

### 6. `agent_pane_width` Rust-side schema verification
`src/workspace.ts` persists `layout.agent_pane_width` via
`mergeWorkspaceState`. If `src-tauri/src/workspace_state.rs` doesn't have
that field on its `Layout` struct (and isn't using
`#[serde(deny_unknown_fields = false)]`-style permissiveness), the value
gets dropped on every round-trip and the agent pane re-opens at the
default 640px on every launch.

Fix:
* Open `src-tauri/src/workspace_state.rs`, find the `Layout` struct, add
  `agent_pane_width: Option<u32>` (or `f64`).
* Confirm by editing the agent-pane width, restarting the app, verifying
  the new width persists.

Acceptance check:
* Drag agent pane to 800px. Quit app. Relaunch. Width is still 800px.

## Low impact (polish)

### 7. Tab context menu polish
Commit `25364e5`. Issues:
* No `Esc` key to dismiss (only backdrop click).
* Pure mouse-only: no keyboard alternative for power users.
* Inline `style.X = ...` instead of using a `.tab-context-menu` CSS class
  even though the className is set.
* Doesn't reposition near viewport edges — can render partly off-screen
  on a tab near the right or bottom.

Fix:
* Move all inline styles into `styles.css` under `.tab-context-menu` and
  `.tab-context-menu-item`.
* Add `Escape` keyboard listener that calls `close()`.
* Before final positioning, check
  `e.clientX + menu.offsetWidth > window.innerWidth` and clamp; same
  for vertical.

Acceptance check:
* Right-click a tab pinned at the far right of the strip — menu opens
  fully on-screen.
* `Esc` dismisses the menu.

### 8. Editor font-size fallback default mismatch
* `src/editor.ts:86`: `var(--editor-font-size, 13px)`
* `src/file-editor.ts:235`: `var(--editor-font-size, 12px)`
* `src/settings.ts`: `DEFAULT_SETTINGS.editorFontSize = 12`

Harmless once the CSS var is applied (which happens on `SettingsManager`
construction), but the fallbacks disagree. If the var ever fails to set
(e.g. a user clears localStorage and the manager doesn't re-apply
synchronously), the input bar editor and the file editor render at
different sizes.

Fix:
* Pick one default (12). Use it as the fallback in both files.

### 9. Pane-visibility state not persisted across launches
`previewVisible`, `terminalVisible`, `agentVisible` are runtime fields
that default to `true` on every workspace boot. If the user closes the
agent pane, restarts, the agent pane is open again.

Fix:
* Add to `LayoutPrefs`: `preview_visible: boolean`, `terminal_visible: boolean`,
  `agent_visible: boolean`.
* Hydrate from `state.json.layout` in `hydrateWorkspaceState`.
* Persist via `persistLayout` whenever a `toggle*` runs.
* Mirror the field on the Rust side.
* Decide: are these per-project (state.json) or global (settings.ts /
  localStorage)? Layout dragging is per-project today; visibility could
  match.

### 10. Theme UI commented out, theme plumbing still exists
Commit `1adfeb5` hid the Appearance & Theme selector since light theme
isn't supported. But `AppSettings.theme` and `getTheme/setTheme` still
exist in `settings.ts`.

Fix (pick one):
* Remove the theme field, getter, setter — there's only one mode.
* Or: leave it, but rename `theme` to `_theme` / mark deprecated and add
  a comment explaining the UI is hidden until light mode lands.
