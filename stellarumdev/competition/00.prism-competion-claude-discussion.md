Prism's Position on the Nine IDE Vectors
Version 2 — Updated May 2026
Prism is an AI-assisted development environment built on the principle that deterministic verification must precede LLM interpretation. This document maps Prism's architecture, capabilities, and design choices against the nine competitive IDE vectors.

Status: Pre-release. In active daily use by the founder, operating in tandem with Google Antigravity. Prism audits, corrects, and verifies Antigravity's output. Antigravity executes tasks from the master plan. Prism runs the quality layer.


Vector 01: Edit Transparency & Diff Quality
Layer: DX | Score: 5 | Status: Definitive — market-leading
Prism's approach
Prism renders line-level, truthful diffs correlated with actual disk state. Every edit operation (write, delete, move, mkdir) is logged in a timeline with semantic labeling:

Truthful Write Artifacts: The diff card shows exactly what was written to disk, with precise line counts and no trailing newline phantoms
Operation Taxonomy: Non-edit operations are properly labeled as delete, move, mkdir rather than defaulting to generic "edit" labels
Reverting Granularity: Full one-click undo per operation; approval flow for all writes with reject feedback looped to the agent
Agent Action Log: Every substrate cell call (typecheck, ast_query, http_fetch, etc.) is visible with actual command output and exit codes — not the model's paraphrase
Execution-Level Verification: Unlike Antigravity's outcome-level Artifacts (screenshots, recordings), Prism's diff is correlated with raw tool call outputs and exit codes — a fundamentally more rigorous verification model for senior developers

Competitive advantage
Browser-based generators (Bolt.new, Lovable, Base44) collapse the edit loop into "accept/reject whole file" — no line-level transparency. Cursor offers in-editor diffs but anchors to the editor UX, not a separate audit trail. Antigravity provides outcome Artifacts (did it work visually) rather than execution Artifacts (what exactly ran and what did the compiler say). Prism uniquely provides professional-grade audit transparency as a first-class feature.
The milestone: Prism designed a five-phase refactor plan for the diff card feature, executed it autonomously across all five phases with no assistance beyond approval cards. The product demonstrated its own thesis on itself.

Vector 02: Agent Reliability & Coherence
Layer: Capability | Score: 4 | Status: Above market — substrate-backed guarantee
Prism's approach
Agent reliability is operationalized as coherence between claims and evidence:

Multi-Pass Verification: "Always Verify" mode enables a second supervisor model that independently fact-checks every response
Structural Guards: Default "Auto Verify" enforces the Grounded-Chat protocol (SOURCE → EVIDENCE → WORKING) via automated rigor scanning
Verification-by-Default Philosophy: The substrate decides what counts as proof; the LLM cannot raise a finding above candidate confidence alone
Approved Writes Only: Every write tool call goes through explicit user approval before touching disk
Error Recovery Protocol: Model sees tool-result messages on rejections and reacts adaptively
Self-Correcting Loop: In production use, Prism catches and flags Antigravity's errors; when Antigravity fixes what Prism identifies, the fix is logged in an AI Review file with a structured assessment of what was missing and why the corrected implementation is more robust

Competitive advantage
Most AI coding tools report findings with overconfident phrasing because the model has no other way to express itself. Prism downgrades LLM-only claims automatically:

Grep-only finding → [candidate warning] (regardless of tone)
AST-backed finding → [probable]
Typecheck/LSP/test/runtime-backed finding → [confirmed error]

The grader is deterministic (gradeFinding in findings.ts) and refuses to be influenced by the model's confidence language.

Vector 03: Context Awareness & Codebase Understanding
Layer: Capability | Score: 4 | Status: Deterministic + optional depth
Prism's approach
Context awareness is grounded in real project structure, not hallucination-prone retrieval:

Whole-Repo Indexing: Zero automated indexing by default (avoids phantom-file hallucinations)
User-Curated Context: @file syntax to explicitly attach files the agent sees; no surprise background inference
Symbol/Dependency Awareness: ast_query substrate cell resolves identifiers via the TypeScript compiler API; verifies if a symbol exists and traces its declaration
LSP Grounding: Integration with rust-analyzer, pyright, gopls, typescript-language-server for type-aware diagnostics
Persistent Memory: Each workspace has a .prism/state.json spine that survives tab reload and app relaunch; agent references prior audit/build context
Multi-Tab Cross-Project Context: Independent tabs mean a developer can ask Prism to examine how embedded vector search was implemented in Project A, learn from it, and apply the pattern to Project B — all in one UI session
Skill Awareness (see Vector 08): When Skill Awareness is ON, the LLM actively monitors the skill folders and proactively suggests applicable skills mid-task — turning prior codeified knowledge into live context


Vector 04: Interaction Model & Conversation Quality
Layer: DX | Score: 4 | Status: Terminal-first + IDE affordances
Prism's approach
Interaction is layered — the terminal is always available; the IDE surface is optional:

Slash Commands: /audit, /fix, /build, /refactor, /new, /test-gen, /models, /help, /history, /save, /load etc. (deterministic, no LLM parsing)
Per-Tab Isolation: Each tab is a fully separate shell + chat + memory; model can be switched mid-conversation; multiple projects open simultaneously in one UI instance
Model Switching: Flip models per turn (careful reasoning with Haiku in one tab, quick spot-fixes with Flash in another)
Inline Chat + Terminal Block Output: Agent responses render with syntax highlighting, inline diff cards (collapsible, Cyber-Noir aesthetics), code blocks with high contrast
File Tree + Editor Overlay: CodeMirror buffer with line numbers, undo/redo, diagnostic squiggles, gutter markers
File Reader (Full-Screen / Side-by-Side): Any project file opens into a large full-screen or dual-column view — built for reading, not compressed into a narrow sidebar box
Problems Panel: Severity + confidence filtering; click a finding to see inline code snippet
Comparative Reader / Cinema Mode: Large-format, dual-column floating modal for side-by-side code analysis
Persistent Session Management: Custom tab titles, chat export to markdown, round-trip load from saved chats
Natural Language File Operations: The terminal handles natural language instructions for creating, editing, and moving everyday files — not just code. This makes Prism useful across the full range of a developer's actual work


Vector 05: Code Generation Quality & Accuracy
Layer: Capability | Score: 3 | Status: Substrate-verified
Prism's approach
Code generation is not autonomous — it is planned → generated → verified → iterated:

Correctness on First Attempt: Model choice (GPT-5.4, DeepSeek, Codestral) dramatically affects this; Prism is model-agnostic and routes to the right model for the task
Idiomatic Style Matching: The agent reads your existing codebase via @file to infer style; enforces TypeScript conventions, naming patterns, etc.
Test Generation: /test-gen mode scaffolds tests for a named function, with substrate verification (tests must pass)
Boilerplate Elimination: /build mode accepts high-level intent and generates minimal, correct code
Mandatory Verification Loop: After generation, typecheck runs, http_fetch probes routes, test suite runs. If any step fails, the model sees the error and iterates. Most other tools generate and hope.

Note
Code generation quality is heavily model-dependent and model-agnostic by design. Prism's edge is the verification loop, not the raw generation quality.

Vector 06: Debugging & Code Analysis
Layer: Capability | Score: 4 | Status: Systematically strong — underinvested market-wide
Prism's approach
Debugging is substrate-backed and systematically supported:

Error Message Interpretation: The agent sees actual compiler errors (typecheck output), stack traces (from run_tests), and HTTP status codes
Stack Trace Traversal: Traces from error message to source file via file_snippet (line-windowed reader) or full file read
Static Analysis Integration: LSP servers (rust-analyzer, pyright, gopls) surface cross-file lints, unused imports, dead code
Live Runtime Error Capture: http_fetch and e2e_run substrate cells capture actual HTTP failures, timeouts, assertion mismatches
Security Vulnerability Detection: LSP servers report when configured; substrate extensible to run SAST tools

Competitive advantage
Generating code is ~30% of professional dev time; debugging is ~70%. The market optimises for generation because benchmarks measure it. Prism's substrate cells surface runtime errors (not simulations) directly to the agent — a missing env var, wrong port, stale import, broken dependency — all seen by the agent as real tool output, not paraphrased by the model.

Vector 07: Deployment & Environment Integration
Layer: Capability | Score: 2 | Status: Terminal-native, not end-to-end
Prism's approach

In-IDE Deployment: Not yet built; roadmap item
Browser Preview / Live Reload: Works if dev server is running; http_fetch probes it
Git / Version Control: Full shell access; agent runs git commands, reads history, inspects diffs
CI/CD Pipeline: prism-audit CLI designed to drop into GitHub Actions; emits annotations (--format=github)
Cloud Provider Integrations: Not yet; requires new substrate cells

Honest assessment
Bolt.new, Lovable, and Base44 win this vector. Prism doesn't close the "generate → ship" loop in one gesture. The deliberate bet: terminal-native design means any deployment tool (Vercel CLI, Netlify CLI, Docker, Terraform) can be wired in. The agent is not sandboxed and can run actual deploy scripts. Flexibility over convenience.

Vector 08: Customisation & Extensibility
Layer: Fit | Score: 5 | Status: Definitive — skill system is the core differentiator
Prism's approach
Skill System — the knowledge operating system
Prism's Skills architecture is the most consequential extensibility feature in the market. Unlike plugin systems that add features, Skills codify knowledge:

Skill Awareness (ON/OFF): When enabled, the LLM actively monitors skill folders and proactively suggests applicable skills when the current task could benefit. It also surfaces alternatives. This turns accumulated institutional knowledge into live, ambient context.
Skill Builder: A fast-path for converting a long, high-value conversation (where much was learned) into stored, reusable, deployable knowledge. The output is a skill file that can be invoked in future sessions.
Compound Leverage Example: "Research a16z, design a template following how they make pitchbooks, create an a16z skill, review the Prism codebase, then use the a16z skill to write the Prism pitchdeck" — executable in one session, reusable indefinitely.
Skill Library: Growing collection of domain skills (Claude Code Protocol, VC pitch framing, project protection, feature catalog, UI search patterns, etc.) visible in the file tree

Additional extensibility

Custom AI Model Routing: /model <name> to switch per turn; ~20 curated options via OpenRouter (GPT-5.4, Kimi, DeepSeek, Grok, Codestral, Haiku, Flash, etc.)
Keybinding / UI Personalisation: Resizable layout with persistent state; custom session titles; theme support (Cyber-Noir, oneDark)
Custom Prompt Templates / Personas: Config file support; system prompts per consumer mode designed to be overridable
Self-Hosted: No dependency on Prism servers; all processing is local or via OpenRouter API; substrate cells run against local project
CLI Layer: prism-audit has no LLM dependency at all — the substrate is independently useful

Why this matters
The vision: Prism is an LLM on steroids. Not just a better coding assistant — a system where every piece of accumulated knowledge is codified, stored, and re-deployable on future tasks. The skill system is the mechanism that converts one-time insights into durable leverage.

Vector 09: Target Fit & Audience Alignment
Layer: Fit | Score: 3 | Status: Senior dev / verification-first
Prism's approach
Audience is deliberately scoped:

Beginner / No-Code Friendliness: Low (CLI is intentionally demanding; slash commands have a learning curve; substrate traces require reading)
Senior Dev Power-User Ceiling: Highest in market (full terminal access, model switching, AST-level introspection, audit trails, chat export, skill system)
Team / Collaborative Features: Not yet (roadmap: shared workspaces, chat threading, team audit reports)
Pricing Model: Pay-per-API-call via OpenRouter (transparent, no subscription lock)
Platform Availability: macOS desktop (Tauri app) + headless CLI (prism-audit for CI); Windows/Linux on roadmap

Self-positioning

Bolt.new, Lovable: 5 on beginner / 1 on senior dev ceiling
Cursor: 3 on beginner / 4 on senior dev ceiling
Prism: 2 on beginner / 5 on senior dev ceiling

This is intentional. The tradeoff is not a flaw — it is the product identity. Prism targets full-stack engineers building production systems, teams that care about audit trails and reproducibility, and developers who want to understand and correct what the AI is doing.

Architectural foundation
Substrate first, LLM second. Seven deterministic cells (typecheck, ast_query, run_tests, http_fetch, e2e_run, lsp_diagnostics, file_snippet) run before any model interpretation. Findings are graded by source (confirmed > probable > candidate), not by LLM confidence language.
Design tradeoffs

Speed vs. Correctness: Latency increases because substrate calls shell out to real compilers. Intentional.
Convenience vs. Transparency: No auto-indexing; files attached explicitly via @file. No hallucinated context.
Breadth vs. Depth: Each consumer (audit, build, fix, refactor) is 600–1,000 lines. Easy to extend.

Production use (as of May 2026)
Prism operates daily in tandem with Google Antigravity. Antigravity executes; Prism audits and corrects. When Antigravity fixes a Prism-identified issue, the resolution is logged in a structured AI Review file — building an institutional record of what the AI got wrong and what made the fix more robust. This human-AI-AI workflow is the most advanced in the market and is not replicated by any competitor.

Verification Pass (~45min): Independent supervisor model fact-checks each response
Auto Model Routing (~2h): Reasoning → Claude, code → Codestral, web → Sonar, vision → Flash
