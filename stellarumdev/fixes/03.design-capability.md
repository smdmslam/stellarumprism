## Vision

Transform Prism from a verification-first coding agent into a **taste-first design partner**.  
When a developer says “design a luxurious AI SaaS dashboard” or “make this landing page feel premium and 2026-native”, Prism should draw from a personally-curated, high-signal library of references (Midjourney gallery, v0 projects, Awwwards, Behance, personal critique notes) and generate UI that reflects refined aesthetic judgment rather than generic LLM tropes.

This is not just another “generate UI” feature. It is the second pillar of the Skill System:
- Skills = codified process & knowledge
- Design Library = codified taste & visual language

Together they give Prism a permanent memory of *how* we like to build software *and* *how* we like it to look.

## Why This Matters for Prism’s Competitive Position

From the competition matrix (`prism-competion-claude-discussion.md`):

- **Vector 05 – Code Generation Quality**: Currently scored 3 (“Adequate — model-agnostic by design”). The design library raises this to 5 by grounding generation in curated taste.
- **Vector 08 – Customisation & Extensibility**: Already a 5 thanks to the skill system. The design library makes taste as reusable as process.
- **Vector 03 – Context Awareness**: The library becomes rich, structured context that Skill Awareness can proactively surface.

This feature directly addresses the “fabricated aesthetic” problem we have seen with weaker models (Gemini Flash Lite inventing features or producing generic UIs). With the design library + Grounded-Chat rigor, Prism will cite real references and synthesize rather than hallucinate style.

## Metadata Schema (per reference)

Every entry must contain:
- `source`: midjourney | v0 | awwwards | behance | personal
- `original_prompt` (when available)
- `design_notes` / critique (why it has taste)
- `tags`: asymmetric-layout, variable-fonts, glassmorphism, micro-animations, high-contrast, neo-brutalism, etc.
- `dominant_palette`: hex values + names
- `typography`: pairings, scale, weights
- `motion_language`: subtle, energetic, cinematic, etc.
- `why_it_has_taste`: 2–4 sentence human judgment
- `sector`: primary bucket
- `visual_priority`: 1–5

## Core Skill: design-core.md

This will be the master skill. When Skill Awareness is ON, Prism will:
1. Detect design-related intent.
2. Load relevant sector indexes + 2–4 closest examples.
3. Synthesize a *new, non-derivative* aesthetic direction.
4. Always cite which library pieces influenced each decision (typography, layout rhythm, color usage, motion).
5. Generate both moodboard (vision model) and production code (React + Tailwind or whatever the project uses).

New slash command: `/design <description> [--reference <sector>]`

## Success Criteria

- Generated designs visibly reference the library (citations in output).
- No more generic gradients, Inter UI, or “AI slop” aesthetics unless deliberately requested.
- User can evolve their taste over time simply by adding new examples and notes to the library.
- The system feels like an extension of *your* design sensibility (not Midjourney default or Claude’s generic taste).
