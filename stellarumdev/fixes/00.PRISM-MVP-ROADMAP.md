## Gate 1: Core Quality & UX (The "Utility" Gate)
*Focus: Ensure Prism is a "Verified" daily driver.*

- [ ] **1.1 Substrate Quality Audit**: Run the agent through "Ideation-to-Completion" end-to-end tests. Identify and fix edge cases in `/build` and `/audit`.
- [ ] **1.2 Friction Removal Pass**: Audit every slash command and UI interaction. If a task takes > 2 clicks or > 5 seconds of thinking, it’s a friction point.
- [ ] **1.3 Simplified Directory Entry**: **Partial.** Bookmarks exist (`/bookmark`, sidebar bookmarks list, onboarding can add roots, Settings auto-bookmark toggle). **Remaining:** richer prompt-area autocomplete for recent paths if needed.
- [x] **1.4 Approval Audit Trail**: Implement the backend logger that records every user approval (timestamp, file, diff snippet) into `audit-trail.jsonl`. **(COMPLETED)**
- [x] **1.5 Onboarding Wizard**: First-run flow for project directory, API key path, and primary model — **shipped** (premium wizard UI). Developers can re-open via `/onboarding` for testing.

## Gate 3: Intelligence & Habits (The "Memory" Gate)
*Focus: Making Prism "learn" the user's style.*

- [x] **3.1 Rolling Activity Memory**: Implement the interaction logger that tracks patterns (e.g., "User always runs /audit before /fix"). **(COMPLETED)**
- [x] **3.2 Habit/Memory Storage**: Create the storage area in `prism/habits/` for learned behaviors. **(COMPLETED)**
- [x] **3.3 Habit Management UI**: Add the Settings tab where users can view, promote, or delete learned habits. **(COMPLETED)**
- [x] **3.4 Path/Folder Correction**: Ensure `history` and `settings` are pointing to the correct localized folders (App Data vs. Workspace). **(COMPLETED)**
- [x] **3.5 Dual-Mechanism Habit & Learning Module**: Implement the explicit `/remember <rule>` slash command and the implicit `[HABIT SUGGESTION]` stream interception toast card (Specced in `04.HABIT-AND-LEARNING-MODULE-IMPLEMENTATION.md`). **(COMPLETED)**
- [ ] **3.6 Advanced Aesthetics & Typography**: Elevate UI surfaces, code blocks, admonitions, and CodeMirror hierarchy (Specced in `04.ADVANCED-AESTHETICS-AND-TYPOGRAPHY-SPEC.md`).

## Gate 4: Business Infrastructure (The "Commercial" Gate)
*Focus: Authentication, Payments, and Tiers.*

- [ ] **4.1 Identity Layer**: Implement Email and Google Auth (Firebase or similar).
    - **Cloud Bookmark Syncing:** Once authenticated, synchronize the user's bookmarks (historically stored locally in `config.toml`) to Firestore under their user account (`/users/{uid}/bookmarks`), allowing bookmarks and directory shortcuts to follow users across devices automatically.
- [ ] **4.2 Subscription State Machine**: Define the tiers (Free, Pro, Enterprise) and the gating logic in the UI.
- [ ] **4.3 Stripe Integration**: 
    - Checkout session flow.
    - Webhook handling (activation/cancellation).
    - Customer portal for managing billing.
- [ ] **4.4 Credits/Token Dashboard**: **Partial.** Toolbar fuel gauge + per-tab token/cost row shipped. **Remaining:** fuller subscription/billing dashboard when Gate 4 lands.

## Gate 5: Distribution & Operations (The "Launch" Gate)
*Focus: Shipping the binary and watching it work.*

- [ ] **5.1 Legal & Terms**: Implement the "Terms and Conditions" modal on first startup (Gate 0).
- [ ] **5.2 Distribution Method**: 
    - Sign and notarize the Mac binary.
    - Create the DMG installer with the "Drag to Applications" modal.
- [ ] **5.3 Update Mechanism**: Implement the in-app "Update Available" notification system.
- [ ] **5.4 Observability (MISSING)**: Integrate **Sentry** (for crash reporting) and **PostHog** (for feature usage analytics).
- [ ] **5.5 Feedback Loop (MISSING)**: Add a "Report Issue" button in the agent pane to capture logs and user feedback for "bad turns."
- [ ] **5.6 Landing Page**: Finalize hero copy, screenshots, demo video, and testimonials.
