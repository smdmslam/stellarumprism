# PRISM Detail Spec 02: IP Isolation & Beta Guardrails
**Date:** 2026-05-05  
**Project:** StellarumPrism  
**Context:** Formalizing the separation between Product and Development data.

---

## 1. The Workspace vs. Intelligence Split
To protect proprietary research while maintaining a functional user workspace, Prism adopts a bifurcated `.prism` directory structure.

### A. The Shipped `.prism` (Product Contents)
These folders are "Project Context" files that live alongside the user's code. They are safe and necessary for the user experience.

*   **`/.prism/skills/`**: The local library of agent skills the user has enabled.
*   **`/.prism/history/`**: User conversation history (visible for `grep` and transparency).
*   **`/.prism/audit/`**: Reports generated by `/audit` and `/review`.
*   **`/.prism/habits/`**: Memory area for project-specific learned habits.
*   **`/.prism/snapshots/`**: Automatic pre-refactor state backups.

### B. The Development `.prism` (Proprietary Musings)
These files are the "Intel" and must be shielded from Git history and user distribution.

*   **`/.prism/intel/`** (or `/.prism/dev/`):
    *   `fixes/`: Implementation and Master plans.
    *   `projects/`: Competitive analysis and strategy docs.
    *   `backups/`: Manual research backups.

### The Mental Picture Flow
| **Layer** | **Location** | **Status** | **Purpose** |
| :--- | :--- | :--- | :--- |
| **Product Logic** | `src/`, `src-tauri/` | **SHIPPED** | The engine. Compiled Rust/Minified JS. |
| **Workspace State** | `.prism/skills/`, etc. | **SHIPPED** | User project data. |
| **Strategic Intel** | `.prism/intel/` | **SHIELDED** | Proprietary research and plans. |

## 2. The "Self-Gaze" Problem & Industry Standards
How Prism protects its own integrity and IP when a user or agent points the tool at its own installation directory.

### 1. The Package Standard (Mac OS)
*   **Binary Compilation**: Rust code is compiled into machine code. It is not readable by humans or agents.
*   **Asset Bundling**: Frontend code is minified and obfuscated during the build process.

### 2. Standard Practice: Explicit Exclusion
The build process (`npm run tauri build`) must be configured to only grab distribution-ready files.
*   **Excluded**: `.git/`, `node_modules/`, and the `.prism/intel/` development folders.
*   **Result**: The shipping DMG contains a "Clean Room" version of the app.

### 3. Agent Guardrails (The Self-Inspection Filter)
Following the standard set by Cursor and VS Code, the Prism agent must be prevented from inspecting its own binaries.

*   **Implementation**: A **Path Guard** in the Rust backend.
*   **Rule**: If an agent command (`read_file`, `grep`, `list_dir`) targets a path starting with the Prism installation directory (e.g., `/Applications/Prism.app`), the backend must return:
    > `Access Denied: Agent cannot inspect internal application binaries.`

---

## 3. The Beta Flow (Mental Model)
1.  **Development**: Full source + history + `.prism/intel` musings.
2.  **The Build**: Compile Rust → Minify JS → **Filter out** all research/agent history.
3.  **The Shipping**: Beta User gets a "Clean Room" tool that knows how to help them, but knows nothing about your internal implementation strategy.
