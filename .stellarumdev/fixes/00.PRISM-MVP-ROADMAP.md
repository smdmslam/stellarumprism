# PRISM MVP Roadmap (Gate-to-Market)
**Date:** 2026-05-05  
**Project:** StellarumPrism  
**Objective:** Sequence all remaining tasks to reach a shippable Beta/MVP.

---

## Gate 1: Core Quality & UX (The "Utility" Gate)
*Focus: Ensure Prism is a "Verified" daily driver.*

- [ ] **1.1 Substrate Quality Audit**: Run the agent through "Ideation-to-Completion" end-to-end tests. Identify and fix edge cases in `/build` and `/audit`.
- [ ] **1.2 Friction Removal Pass**: Audit every slash command and UI interaction. If a task takes > 2 clicks or > 5 seconds of thinking, it’s a friction point.
- [ ] **1.3 Simplified Directory Entry**: Implement the dropdown/autocomplete for recent paths and a "Bookmark" system for common project roots.
- [x] **1.4 Approval Audit Trail**: Implement the backend logger that records every user approval (timestamp, file, diff snippet) into `audit-trail.jsonl`. **(COMPLETED)**
- [ ] **1.5 Onboarding Wizard**: A first-run flow that guides the user through:
    - Setting the initial project directory.
    - Connecting an API key or signing in.
    - Setting a "Primary Model."

## Gate 2: Platform Curation (The "Trust" Gate)
*Focus: Pruning choice and ensuring the app feels "Premium".*

- [ ] **2.1 Workspace/Intel Separation**: (REMOVED - Dev friction too high. All tracked in .stellarumdev)
- [ ] **2.2 Proprietary File Shielding**: (REMOVED - Use system prompt for agent instruction instead of hardcoded Rust blocks.)
- [ ] **2.3 Model Registry Pruning**: Remove underperforming reasoning models (Gemini Pro, etc.) from the user dropdown to simplify choice and improve reliability.
- [ ] **2.4 Self-Inspection Guardrail**: (REMOVED - Simple is better.)

## Gate 3: Intelligence & Habits (The "Memory" Gate)
*Focus: Making Prism "learn" the user's style.*

- [ ] **3.1 Rolling Activity Memory**: Implement the interaction logger that tracks patterns (e.g., "User always runs /audit before /fix").
- [ ] **3.2 Habit/Memory Storage**: Create the storage area in `.prism/habits/` for learned behaviors.
- [ ] **3.3 Habit Management UI**: Add the Settings tab where users can view, promote, or delete learned habits.
- [ ] **3.4 Path/Folder Correction**: Ensure `history` and `settings` are pointing to the correct localized folders (App Data vs. Workspace).

## Gate 4: Business Infrastructure (The "Commercial" Gate)
*Focus: Authentication, Payments, and Tiers.*

- [ ] **4.1 Identity Layer**: Implement Email and Google Auth (Firebase or similar).
- [ ] **4.2 Subscription State Machine**: Define the tiers (Free, Pro, Enterprise) and the gating logic in the UI.
- [ ] **4.3 Stripe Integration**: 
    - Checkout session flow.
    - Webhook handling (activation/cancellation).
    - Customer portal for managing billing.
- [ ] **4.4 Credits/Token Dashboard**: Real-time display of user "Fuel" (remaining balance) based on actual usage.

## Gate 5: Distribution & Operations (The "Launch" Gate)
*Focus: Shipping the binary and watching it work.*

- [ ] **5.1 Legal & Terms**: Implement the "Terms and Conditions" modal on first startup (Gate 0).
- [ ] **5.2 Distribution Method**: 
    - Sign and notarize the Mac binary.
    - Create the DMG installer with the "Drag to Applications" modal.
- [ ] **5.3 Update Mechanism**: Implement the in-app "Update Available" notification system.
- [ ] **5.4 Observability (MISSING)**: Integrate **Sentry** (for crash reporting) and **PostHog** (for feature usage analytics).
- [ ] **5.5 Feedback Loop (MISSING)**: Add a "Report Issue" button in the agent pane to capture logs and user feedback for "bad turns."
- [ ] **5.6 Landing Page**: Finalize hero copy, screenshots, demo video, and testimonials.

---

## Critical Sequential Path
1. **Gate 1 & 2** are required for **Private Alpha** (you + trusted friends).
2. **Gate 3 & 4** are required for **Public Beta** (monetization starts).
3. **Gate 5** is required for **General Release**.
